# ESLint Configuration

## Disabled Rules

The following ESLint rules are disabled for specific use cases:

### `@typescript-eslint/no-unsafe-*` rules
- **Where**: `product.service.ts`, `contentful.adapter.ts`, `reports.controller.ts`
- **Why**: Intentional use of `any` types for:
  - Cache manager store access (no TypeScript definitions available)
  - Contentful API responses (dynamic structure)
  - Report strategy return types (flexible output)

### `@typescript-eslint/unbound-method`
- **Where**: Test files (`*.spec.ts`)
- **Why**: Jest mocks require method references without binding

### `@typescript-eslint/require-await`
- **Where**: `product.controller.ts`
- **Why**: Controller methods return promises from service calls
